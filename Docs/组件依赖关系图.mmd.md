# 组件依赖关系图（Mermaid 索引）

当前组件依赖图已经拆分为两个纯 Mermaid 源文件，便于直接导出 SVG / PDF：

- 高层图：[组件依赖关系图-高层.mmd](</Users/janlor/workspace/GitHub/AIDataInsight-iOS/Docs/组件依赖关系图-高层.mmd>)
- 详细图：[组件依赖关系图-详细.mmd](</Users/janlor/workspace/GitHub/AIDataInsight-iOS/Docs/组件依赖关系图-详细.mmd>)

可直接使用导出脚本：

- [export-mermaid-diagrams.sh](/Users/janlor/workspace/GitHub/AIDataInsight-iOS/scripts/export-mermaid-diagrams.sh)

使用方式：

```bash
chmod +x scripts/export-mermaid-diagrams.sh
./scripts/export-mermaid-diagrams.sh
```

脚本依赖 `mermaid-cli`：

```bash
npm install -g @mermaid-js/mermaid-cli
```

维护建议：

- `.mmd` 是唯一应被人工编辑的源文件
- `.svg` 和 `.pdf` 由脚本导出，不建议手工改动
- 提交代码时建议将 `.mmd`、`.svg`、`.pdf` 一起提交，避免图源与导出产物不一致
- 当 `Package.swift`、模块目录或依赖边发生变化时，应同步更新对应图文件

说明：

- `ModuleAI` 在 `Package.swift` 中直接依赖 `LibraryCommon` 聚合产物，因此详细图中展开成其包含的 target，便于维护时看清真实边界。
- `SSEClient` 当前位于 `Packages/library-basics/Sources/Networking`，因此被归入 `Networking`。
- 如果后续 App 壳工程直接引入更多 package，可在图中的 `App Shell` 节点继续补边。

# AIDataInsight iOS 架构设计说明书

## 1. 文档目的

本文档说明当前 iOS 端的真实架构：App 壳工程、Swift Package 模块划分、`ModuleAI` 的 `Domain / Application / Repositories / Presentation` 分层，以及为 Android / Web 迁移做的准备。

iOS 是当前最完整的参考实现，但跨端生成的源事实是根目录 `docs/cross-platform/contracts`，不是 UIKit 页面、Cell 或 ViewData。

## 2. 总体结构

```text
AIDataInsight App
  -> library-common / AppMain
  -> module-ai
  -> library-common
  -> library-basics
```

目标：

- App 壳工程只负责启动、环境和包装。
- 业务能力沉到 Swift Package。
- `ModuleAI` 按四层组织。
- UseCase 输出跨端 application output，不返回 UIKit 展示模型。
- API path 放在本模块 API descriptor 中，不在 Repository / ViewModel 拼完整 URL。

## 3. 模块职责

### App 壳工程

路径：`app-ios/AIDataInsight`

负责 `AppDelegate`、`SceneDelegate`、scheme、xcconfig、AppIcon、LaunchScreen、Info.plist，并通过：

```swift
Application.agent.frontModule("AppMain")
```

启动模块化入口。

### library-common

路径：`app-ios/Packages/library-common`

负责通用业务模块和协议：

- `AppMain`：根据登录态切换 root
- `Login` / `Privacy` / `Setting`
- `LoginProtocol` / `PrivacyProtocol` / `SettingProtocol` / `ProtocolAI`
- `CommonViewModel` / `CommonRequester`

登录成功后的入口链路：

```text
AppMain
-> Router.target(to: ProtocolAI.self)
-> ModuleAIRouter
-> ContainerViewController
```

### library-basics

路径：`app-ios/Packages/library-basics`

负责基础设施：

- 账号会话：`Account` / `AccountProtocol`
- 模块启动：`AppLaunch`
- 环境配置：`Environment`
- 网络层：`Networking`
- 基础 UI / 工具 / 路由 / 存储

默认 mock baseURL 位于：

```text
Packages/library-basics/Sources/Environment/AppEnvironmentValues.swift
```

### module-ai

路径：`app-ios/Packages/module-ai`

当前是 AI 数据分析业务核心模块，负责 AI Chat、History、函数调用分析、动态参数、图表和流式回复。

## 4. ModuleAI 当前分层

```text
ModuleAI
├── Domain
├── Application
├── Repositories
└── Presentation
```

### Domain

职责：

- 领域模型
- Repository 协议
- 子域 endpoint 语义

代表文件：

- `Domain/AIChat/AIChatRepository.swift`
- `Domain/AIChat/TemplateModel.swift`
- `Domain/AIChat/FunctionName.swift`
- `Domain/AIChat/FunctionArguments.swift`
- `Domain/AIChat/AIChatEndpoint.swift`
- `Domain/History/HistoryRepository.swift`
- `Domain/History/HistoryModel.swift`

约束：

- 不依赖 UIKit。
- 不关心 Cell / ViewData / ViewController。
- `AIChatEndpoint.streamPath` 属于 AI Chat 子域，不属于全局环境配置。

### Application

职责：

- UseCase
- application output
- 业务流程编排
- 稳定跨端输出

代表 UseCase：

- AI Chat：模板、历史详情、函数调用、图表数据、流式回复、点赞反馈
- History：分页、删除单条、清空全部

约束：

- UseCase 不返回 UIKit 展示模型。
- UseCase 不依赖 `UIViewController` / `UICollectionViewCell` / iOS ViewData。
- Android / Web 应从契约生成同等 application output。

### Repositories

职责：

- iOS 网络实现
- API descriptor
- DTO 适配
- DTO -> Domain 映射

代表文件：

- `Repositories/AIChat/DefaultAIChatRepository.swift`
- `Repositories/AIChat/ChatApi.swift`
- `Repositories/AIChat/ChartApi.swift`
- `Repositories/AIChat/FunctionResponseDTO.swift`
- `Repositories/History/DefaultHistoryRepository.swift`
- `Repositories/History/HistoryApi.swift`

约束：

- Repository 调用 `ChatApi` / `HistoryApi` / `ChartApi`。
- 不硬编码完整 URL。
- 流式接口走 `ChatApi.stream` + `CommonRequester.requestSSE`。
- `/chat/template` 的内嵌 JSON 字符串在 Repository 层归一化为 `TemplateModel`。

### Presentation

职责：

- UIKit 页面
- ViewModel 状态桥接
- ViewData / Cell / 图表 UI
- 手势、输入、列表刷新和渲染节奏

代表文件：

- `Presentation/App/ContainerViewController.swift`
- `Presentation/AIChat/ViewControllers/AIChatViewController.swift`
- `Presentation/AIChat/ViewModels/AIChatViewModel.swift`
- `Presentation/History/ViewControllers/HistoryViewController.swift`
- `Presentation/History/ViewModels/HistoryViewModel.swift`

约束：

- UIKit 细节不是跨端源事实。
- ViewController 不直接解析 DTO。
- Cell 不参与业务分支判断。
- ViewData 只服务 iOS UI。

## 5. AI Home 主入口

`ContainerViewController` 当前用 UIKit 子控制器实现：

```text
ContainerViewController
├── main: AIChatViewController
└── menu: HistoryViewController
```

核心语义：

- AI Chat 是主内容。
- History 是辅助面板。
- 打开 History 时刷新列表。
- 选择历史后关闭面板，并让 Chat 在原位加载该会话。
- 打开 / 关闭 History 不销毁 Chat 状态。

跨端只继承这些语义，不继承 UIKit child-controller、navigation controller 或 transform 动画细节。

## 6. AI Chat 主链路

### 模板问题

```text
AIChatViewModel
-> LoadTemplateUseCase
-> AIChatRepository.loadTemplate
-> DefaultAIChatRepository
-> ChatApi.template
-> TemplateModel
```

UI 不解析接口字符串；兼容解析规则已沉淀到跨端契约。

### 函数调用与图表

```text
用户输入
-> SendFunctionMessageUseCase
-> ChatApi.function
-> FunctionResponseDTO.toDomainModel()
-> FunctionName + FunctionArguments
-> LoadChartDataUseCase
-> ChartApi.chart
-> HistoryDetailModel
-> AIChatChartBuilder
-> 图表 Cell
```

固定主链路：

```text
FunctionName -> FunctionArguments kind -> /chart/{FunctionName.rawValue} -> ChartPayload
```

### 历史会话加载

```text
History 选择 historyId
-> AIChatViewController.loadConversation
-> LoadHistoryDetailUseCase
-> HistoryApi.detail
-> AIChatHistoryMapper
-> 消息列表刷新
```

### 流式回复

```text
用户输入
-> StreamAIResponseUseCase
-> AIChatRepository.streamMessage
-> ChatApi.stream
-> CommonRequester.requestSSE
-> SSEClient
-> AIChatCell 增量渲染
```

流式渲染策略：

- 网络层快速接收 SSE `data:`。
- UI 层通过 `CADisplayLink` 控制显示节奏。
- 中文按字推进，英文尽量按单词边界推进。
- 避免每个 chunk 都重建整个列表 snapshot。

## 7. History 主链路

```text
HistoryViewController
-> HistoryViewModel
-> LoadHistoryPageUseCase / DeleteHistoryUseCase / DeleteAllHistoryUseCase
-> HistoryRepository
-> DefaultHistoryRepository
-> HistoryApi
-> HistoryListViewDataBuilder
-> HistoryCell
```

History 的分组 ViewData 只服务 iOS 展示；跨端应使用契约中的 History use case / UI state。

## 8. 网络设计要点

网络层位于：

```text
Packages/library-basics/Sources/Networking
```

核心结构：

- `RequestDescriptor`
- `RequestBuilder`
- `NetworkExecutor`
- `ResponseModel`
- `TokenRefreshCoordinator`
- `SSEClient`

边界：

- 子模块声明 path、method、parameters、headers。
- baseURL 来自 Environment。
- token、Org-Id、刷新和会话失效由 Networking / Account 协作处理。
- SSE 是基础网络能力，不放在 `ModuleAI` 私有实现中。

## 9. 跨平台准备状态

当前 iOS 已完成：

- `ModuleAI` 四层分离
- AI Chat use case 化
- History use case 化
- 动态函数参数固化
- 模板问题解析规则沉淀到契约
- AI Home 的 Chat / History 切换语义沉淀到契约
- 关键链路有单元测试保护

跨端源事实：

```text
docs/cross-platform/contracts
```

iOS 的角色是验证真实产品行为、发现契约缺口、提供参考实现。Android / Web 应从契约生成模型和主链路，不从 UIKit 页面反推领域模型。

## 10. 总结

当前 iOS 端已经从早期的 UIKit + ViewModel 直连网络，演进为更适合跨端迁移的结构：

```text
Domain -> Application -> Repositories -> Presentation
```

后续 iOS 如果出现新的跨端事实，应先更新 `docs/cross-platform/contracts`，再同步各端实现。

# AIDataInsight AI Entrypoint

Use this file as the first prompt context for a new AI conversation, a new model,
or a new developer joining the project. It keeps contract-driven work small and
prevents each AI from rereading the whole repository.

## First Command

Run one of these from the repository root:

```sh
scripts/check-contract-alignment.sh all
scripts/check-contract-alignment.sh app-apple
scripts/check-contract-alignment.sh app-web
```

The script prints:

- current contract version
- the target app's consumed contract version
- pending migrations
- the minimal read set for that app

## Required Reading Order

For normal maintenance, read only:

1. `docs/ai-entrypoint.md`
2. `docs/cross-platform/contracts/contract-manifest.yaml`
3. the target app's `contract-alignment.json`
4. pending migration files printed by the check script
5. only the contract files referenced by those migrations
6. the target app code touched by the migration

Read `docs/ai-generation-guide.md` only when generating a new feature, creating
a new platform app, or when the migration says the normal read set is not enough.

## Contract Rules

- Contract files are the source of truth, not any app implementation.
- If a platform implementation reveals a missing cross-platform rule, update the
  contract and migration first, then update apps.
- Do not silently patch client code to hide contract/API errors.
- Do not mark an app aligned until code and tests satisfy the migration acceptance.
- Generated model files must be regenerated by scripts, not manually edited.

## Versioning

The current contract version lives in:

```text
docs/cross-platform/contracts/contract-manifest.yaml
```

Each app records its consumed version in:

```text
app-ios/contract-alignment.json
app-android/contract-alignment.json
app-harmony/contract-alignment.json
app-web/contract-alignment.json
app-apple/contract-alignment.json
```

When a contract migration is implemented for an app:

1. update app code
2. add or update tests
3. update that app's `contract-alignment.json`
4. run `scripts/check-contract-alignment.sh <app>`
5. run the target app's relevant build/test command

## New AI Prompt Template

Use this in a new conversation:

```text
Read docs/ai-entrypoint.md, then run:
scripts/check-contract-alignment.sh <app>

Only read the minimal files printed by the script. Apply pending migrations for
<app>, update tests, update <app>/contract-alignment.json, and summarize the
verification. Do not read the whole docs directory unless the migration requires it.
```

## Multi-Team Workflow

- Contract owners modify `docs/cross-platform/contracts`, migrations, and fixtures.
- Platform teams upgrade their own app by consuming pending migrations.
- Different teams do not need access to other app source code.
- Shared behavior is enforced through contract files, fixtures, migrations, and
  alignment checks.
- Future repository splitting can move `docs/cross-platform/contracts` into an
  independent contracts repository while keeping each app's `contract-alignment.json`.
